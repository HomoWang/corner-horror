# 開發紀錄

## 2026-07-14：雙螢幕操作重整

### 實測發現

- 把來電、劇情、密碼鍵盤與選項放在手機，會迫使玩家反覆低頭，直接破壞大螢幕驚嚇的
  注意力設計；手機必須是盲操作控制器。
- WebSocket 的 `pointerdown` 與 `pointerup` 可能在大螢幕同一動畫幀前一起到達。若只讀取
  最後的布林狀態，快速點擊會被吃掉；主機端因此保留一次至少一幀的 action pulse。
- 原本的投影換算只把光錐移到畫面中央約一半範圍，加上 vignette 後左右邊界幾乎全黑。
  水平投影靈敏度已由 `0.5` 提高至 `0.72`，邊界限制改為 `1.5%–98.5%`，同時保留最低
  邊緣可見度。
- 大螢幕的 Web Audio 不能由手機送來的 WebSocket 訊息解鎖。正式流程需在掃碼後於主機
  點一次「開啟聲音並開始」，之後才完全交給手機中央鍵。

### 現行設計決定

- 手機只顯示一顆大型中央互動鍵；重新對準為底部次要功能。
- 來電、文字、`0317`、左右抉擇與重玩全部顯示在大螢幕。
- 密碼以四次中央鍵依序輸入；結局選擇以光錐指向左右，再按中央鍵確認。
- 左窗第一個主要驚嚇改為全屏恐怖臉、主機尖叫／衝擊、雷聲與手機震動同步觸發。

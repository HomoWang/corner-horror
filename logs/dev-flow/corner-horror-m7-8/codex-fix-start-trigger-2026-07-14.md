# 開始體驗無法由手機觸發

日期：2026-07-14

## 現象

- 主畫面在控制器送出第一筆方向資料後顯示「開始體驗」。
- 手機控制器的「動作」按鈕只更新劇情輸入狀態，沒有呼叫體驗啟動函式。
- 啟動函式先等待主機端 Web Audio 解鎖，才隱藏開始畫面與啟動導演流程；音效無法解鎖時，影像流程可能看起來沒有反應。

## 修正

- 手機在開始／結束疊層可見時按下「開始／動作」，會開始或重玩體驗。
- 啟動時立即隱藏疊層並開始影像劇情，不再等待 Web Audio。
- 主機端 click 仍會在同步使用者手勢內嘗試解鎖音效；手機遠端觸發無法解鎖時，主畫面提示另外按「啟動房間聲音」。
- 手機按鈕與提示文字改為清楚標示「開始／動作」的雙重用途。
- 體驗開始時同步要求手機播放持續的暗黑環境聲；因手機音效已由感測器開始按鈕解鎖，不受主機遠端啟動的瀏覽器限制。
- 主機環境音改為多層低頻拍頻、噪聲移動與不協和泛音，並提高可聽音量與加入動態壓縮。
- 手機來電改為 440/480 Hz 經典電話雙音、三次長響節奏與同步震動；耳語改為雙共振峰噪聲塑形。

## 驗證重點

1. 手機完成感測器授權並送出方向資料後，主畫面顯示「開始體驗」。
2. 不操作主機，直接在手機按「開始／動作」，主畫面應立刻進入體驗。
3. 體驗結束顯示「再玩一次」後，再按同一顆手機按鈕應重新開始。
4. 若主機沒有聲音，點主機左上角「啟動房間聲音」，影像流程仍應持續。
